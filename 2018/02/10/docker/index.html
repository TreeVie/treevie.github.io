获取镜像安装

源地址

  1. 启动docker

    1. systemctrl docker start
    2. 通过docker快捷方式启动(现在是用这个)

  2. 下载centOS7镜像 docker pull centos:7.2.1511
    查看下载的镜像 docker images 或者 docker image ls

  3. 启动centOS7 docker run -it 0a2b /bin/bash

    1. 查看当前系统信息 cat /etc/os-release

    2. 参数说明

      • docker run 是容器运行命命令
      • -it 是两个操作 -i 是交互式操作， -t是终端
      • --rm 这个参数是说容器退出后随之将其删除。默认情况下，为了排障需求，退出的容器并不会立即删除，除非手动 docker rm。我们这里只是随便执行个命令，看看结果，不需要排障和保留结果，因此使用 –rm 可以避免浪费空间
      • 0a2b 是IMAGE ID 只要能区分出需要启动的镜像就OK
      • /bin/bash 放在镜像名后的是命令，这里我们希望有个交互式 Shell，因此用的是 bash
  4. exit退出当前容器

容器操作

启动

  1. 新建一个容器并启动
  2. 将在终止状态（stopped）的容器重新启动

新建并启动

docker run
Docker 在后台运行的标准操作：

  • 检查本地是否存在指定的镜像，不存在就从公有仓库下载
  • 利用镜像创建并启动一个容器
  • 分配一个文件系统，并在只读的镜像层外面挂载一层可读写层
  • 从宿主主机配置的网桥接口中桥接一个虚拟接口到容器中去
  • 从地址池配置一个 ip 地址给容器
  • 执行用户指定的应用程序
  • 执行完毕后容器被终止

启动已终止容器

docker container start容器的核心为所执行的应用程序，所需要的资源都是应用程序运行所必需的。除此之外，并没有其它的资源。可以在伪终端中利用 ps 或 top 来查看进程信息

运行时

后台运行

程序在后台运行 宿主机不需要提供交互窗口 使用-d

上面的命令会返回conteinerID,echo并不会在宿主机打印 hello world 而是在container中打印,可以使用docker logs containerid来查看相应的容器中的打印信息

终止

docker container ls -a查看当前的容器
docker container stop
docker container restart 可以让一个终止态的容器启动，也可以让一个运行态的容器重启

进入容器

attach命令

docker attach 9df4d2fa 9df4d2fa是containerID
这种方式下，在stdin(标准输入)中输入exit会导致容器停止

exec命令

docker exec -it 9df4d2fa bash在stdin中输入exit不会导致容器停止

删除容器

  • docker container rm musing_jang删除NAMES为musing_jang的容器，也可以指定容器id进行删除
  • 清除所有的终止态容器docker container prune

私有仓库

获取私有仓库

  • -d 后台运行
  • -v 将镜像文件存放在本地的指定路径
  • -p 端口映射
  • --restart=always 在容器退出时总是重启容器,主要应用在生产环境
  • --privileged=true 在CentOS7中的安全模块selinux把权限禁掉了，参数给容器加特权，不加上传镜像会报权限错误OSError: [Errno 13] Permission denied: ‘/tmp/registry/repositories/liibrary’)或者（Received unexpected HTTP status: 500 Internal Server Error）错误
  • --name 指定容器的名称

数据卷

创建数据卷

volume create my-vol

  • 查看所有的数据卷 docker volume ls
  • 查看数据卷具体信息 docker volume inspect my-vol

启动一个挂载数据卷的容器

docker run -d -P --name web --mount source=my-vol,target=/webapp training/webapp python app.py

挂载一个本地主机文件作为数据卷

--mount 标记也可以从主机挂载单个文件到容器中

这样容器中输入的命令就可以在当前用户下的.bash_history中查看

删除数据卷

  • docker volume rm my-vol
  • 清理无主的数据卷docker volume prune

网络

映射端口

docker run -d -P --name web training/webapp python app.py

  • -P 将容器中的开放端口映射到宿主机的一个随机端口上

  • -p 配置映射关系

    1. ip:hostPort:containerPort 映射指定地址的指定端口
    2. ip::containerPort 映射指定地址的任意端口
    3. hostPort:containerPort 映射所有接口地址,默认会绑定本地所有接口上的所有地址

  • 查看端口container日志docker logs -f web

  • 查看端口
